현재 브랜치의 MR 설명을 아래 템플릿에 맞춰 작성해줘.

## 사전 작업: 테스트 자동 실행

MR 본문 작성 전에 아래 테스트를 실행하고 결과를 수집한다:

1. 단위 테스트: `npm run test` 실행
2. 결과를 MR 본문 "참고 사항" 섹션에 아래 형식으로 첨부한다:

```
#### 테스트 결과
- 단위 테스트 (Vitest): X passed / Y failed
```

- 테스트 파일이 없으면 "테스트 없음" 으로 표시
- 테스트 실패해도 MR 작성은 계속 진행 (실패 결과 그대로 표시)

---

작성 전에 다음을 먼저 확인해:

1. 현재 브랜치명 (`git branch --show-current`)
2. dev 대비 커밋 목록 (`git log dev..HEAD --oneline`)
3. 변경된 파일 목록 (`git diff dev...HEAD --stat`)
4. 커밋 메시지에서 이슈 번호, 작업 내용 파악

템플릿을 채울 때:

- 체크박스(신규/버그/개선/기타)는 커밋 내용 보고 적절한 항목에 체크
- "개발 사유": 왜 이 작업을 했는지 간결하게
- "수정 사항": 변경된 파일과 핵심 변경 내용을 항목별로 정리
- 연관 이슈 번호는 브랜치명 또는 커밋에서 추출
- 나머지 고정값(담당자, 테스트 방법 등)은 템플릿 그대로 유지
- 최종 결과물은 반드시 아래처럼 코드블록으로 감싸서 출력:
  ````markdown
  [MR 본문 내용]
  ````
